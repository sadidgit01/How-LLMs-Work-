01 Stage

The Prompt

Everything begins with text. A user types a question, a system prompt is prepended, special tokens may frame the conversation. The model receives a single string — nothing more, nothing less.

The context window defines how much text can flow in at once. Modern windows range from 4K to 2M tokens; everything beyond is invisible to the model.

Key insight

The model has no memory. Only what's in the window exists.

02 Stage

Tokenization

Text is split into tokens — chunks that are often words, sometimes sub-words, occasionally single characters. Each token maps to an integer from a fixed vocabulary.

Byte-Pair Encoding (BPE) learns the most frequent character sequences during training, building a vocabulary of 50K–100K tokens. "Understanding" might become under + stand + ing.

Key insight

Tokens ≠ words. A model never sees "words" — only integer IDs.

03 Stage

Embedding

Each integer ID becomes a vector — a list of typically 768 to 12,288 floating-point numbers. This vector is the model's learned representation of what that token means.

Tokens with similar meanings land near each other in this high-dimensional space. "King" minus "man" plus "woman" lands suspiciously close to "queen".

Key insight

Meaning is geometry. Semantics live in 12,288-dimensional space.

04 Stage

Positional Encoding

Transformers process all tokens simultaneously — they have no inherent sense of order. So we add a position vector to each embedding, telling the model "this is token number three".

Modern models use RoPE (Rotary Position Embedding), which rotates pairs of dimensions by an angle proportional to position. This generalizes to longer sequences better than learned absolute positions.

Key insight

Without position info, "dog bites man" equals "man bites dog".

05 Stage

Self-Attention

The heart of the transformer. Every token forms three vectors — Query, Key, Value. Each token's Query is matched against every Key to produce attention weights.

These weights determine how much of each token's Value flows into the output. In "the cat sat on the mat because it was tired", "it" attends strongly to "cat" — that's how the model knows what "it" refers to.

The equation
Attention(Q, K, V) = softmax(QKT / √dk) · V
Key insight

This is the only place tokens interact. Everything else is per-token.

06 Stage

Multi-Head Attention

One attention pattern isn't enough. The model runs attention in parallel across 8 to 96 "heads", each learning to attend to different relationships.

One head might track syntax (subject-verb agreement). Another tracks coreference ("it" → "cat"). Another tracks long-range dependencies. Their outputs are concatenated and projected back to a single vector.

Key insight

Different heads learn different linguistic phenomena — independently.

07 Stage

Feed-Forward Network

After attention mixes information between tokens, each token's vector passes through a small two-layer neural network — independently, position by position. This is where the model "thinks" about what each token means in context.

The hidden dimension typically expands 4× before contracting back. Modern models use SwiGLU or GELU activations instead of plain ReLU. The bulk of a model's parameters — and knowledge — lives here.

The equation
FFN(x) = SwiGLU(x · W₁) · W₂
Key insight

Attention routes information. The FFN is where knowledge actually resides.

08 Stage

Residual & LayerNorm

Wrapped around every attention and FFN sublayer are two essential ingredients: a residual skip connection (which adds the input back to the output) and a layer normalization (which keeps activations stable).

Without residual connections, training deep transformers is impossible — gradients vanish before reaching early layers. LayerNorm prevents activation magnitudes from drifting during training.

Key insight

Skip connections let information bypass any single layer — the model is a stack of refinements.

09 Stage

Layer Stacking

One transformer block does little. Stack 12 to 120 of them, and something remarkable emerges. Each layer refines the representation, building abstraction on top of abstraction.

Probing studies reveal a rough hierarchy: early layers track syntax and part-of-speech, middle layers capture entities and relationships, late layers handle reasoning and task-specific structure. Depth is not just bigger — it's qualitatively different.

Key insight

Intelligence isn't a single mechanism — it's the same mechanism, repeated, refining at every level.

10 Stage

Output Projection

The final hidden state — a single vector representing the model's understanding of the sequence so far — is multiplied by the unembedding matrix to produce a score for every token in the vocabulary.

These raw scores are called logits. A 50,000-word vocabulary means 50,000 logits per position. Often the embedding matrix is reused as the unembedding matrix (called "tied embeddings").

Key insight

Every generation step asks 50,000 questions at once: "how likely is each word?"

11 Stage

Softmax & Sampling

Logits become probabilities via softmax. Then the model picks one — and only one — token. How it picks defines the model's "personality": greedy, careful, creative, or chaotic.

Temperature sharpens or flattens the distribution. Top-k keeps only the k most likely. Top-p keeps the smallest set whose probabilities sum to p.

The equation
P(ti) = exp(zi / T) / Σ exp(zj / T)
Key insight

The model never "decides" — it samples. Randomness is a feature, not a bug.

12 Stage

Autoregression

The chosen token is appended to the input. The entire pipeline runs again. And again. And again — one token at a time, until a stop token appears or a length limit is hit.

This is why LLMs are slow at generating text: every output token requires a full forward pass through the entire network. Generating 1000 tokens means running the model 1000 times. KV-caches avoid recomputing attention for past tokens, but the cost remains roughly linear in output length.

Key insight

An LLM doesn't write sentences. It writes one token, then changes its mind about what comes next — one token at a time.
